容器卷

容器之间可以有一个数据共享的技术，docker 容器的数据可以同步到本地，这就是卷技术

卷的设计目的就是数据的持久化，完全独立与容器的生命周期，因此 Docker 不会在容器删除时删除其挂载的数据卷。

特点：

  1. 数据卷可以在容器之间共享和重用数据。

  2. 卷的更改可以直接生效。

  3. 数据卷的更改不会包含在镜像的更新中。

  4. 数据卷的生命周期一直持续到没有容器使用它为止。

容器的持久化

容器间继承+共享数据

总结一句话：容器持久化和同步技术，容器间也是可以数据共享的！
